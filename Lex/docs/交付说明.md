# 编译原理专题实践 — 交付对照说明

本仓库对照**周晓宇**课程大纲整理的「可提交 / 待扩展」清单；实验报告请按 **Lex / Yacc / 中间代码** 分章撰写（课件要求）。

## 1. Lex（SeuLex）

| 课件要点 | 本仓库 |
|----------|--------|
| Lex 输入三段结构 | `c99.l`（全集）；子集规范另见 `c99_subset.l` |
| 由 RE→NFA→DFA→代码 | `src/`、`include/` 下 **seulex** 相关模块（`regex_converter`、`nfa`、`dfa`、`code_generator`、`lex_parser`） |
| 生成 `lex.yy.c` | **`build_parse.cmd` 用 SeuLex 从 `c99.l` 生成**（勿长期提交 Flex 版；旧 Flex 产物仅作对照） |
| 测试 | `test_lexer.exe`、`build_parse.cmd` 中词法冒烟 |

**待扩展（报告可写计划）**：NFA/DFA **Graphviz 可视化**、一键 `seulex c99.l lex.yy.c` 写入 `build_parse.cmd`。

## 2. Yacc（SeuYacc）

| 课件要点 | 本仓库 |
|----------|--------|
| LR(1) First、项闭包、预测符、变迁 | `parser/seu_lr1_parser.cpp` |
| LR(1) 分析表 + 总控 | 同上，`seulr_parse` |
| **LR(1)→LALR(1)** 同心合并 | `compress_lr1_to_lalr()`：合并 ACTION/GOTO 行并重映射移进目标 |
| 产生式来源 | `parser/grammar.cpp`（与 `y.tab.h` 一致）；**完整 .y 解析器**未实现，子集规范见 `c99_subset.y` + `tools/yacc_subset_scan.cpp`（扫描 `%token` / `%%` 演示） |

**答辩表述建议**：「文法以 `grammar.cpp` 为单一真源，与 `c99.y` / `c99_subset.y` 对齐；Yacc 文件解析课程要求的部分由 **子集扫描工具 + 书面产生式** 覆盖，全功能 yacc 解析为后续工作。」

## 3. 中间代码生成

| 课件要点 | 本仓库 |
|----------|--------|
| 语义分析输入为**语法树** | `reduce_prod` 建 `ASTNode`，`c99_parse_main` 打印树 |
| **三地址码骨架** | `semantic/ir_from_ast.cpp`，入口 **`c99_ir.exe`**（`parser/c99_ir_main.cpp`） |
| 带语义节点 / 符号表 / 基本块 / LLVM / Jimple | **骨架已具备遍历与输出格式**；符号表链式作用域、回填、LLVM C++ API、Soot 需按课件第 6–9 周继续实现 |

## 4. 期末提交文件夹建议（课件）

- `c99.l`、`c99.y`（全集）+ **`c99_subset.l`、`c99_subset.y`（子集）**
- 生成器源码：`src/`、`include/`、`parser/`
- 生成结果：`lex.yy.c`、（可选）`c99.tab.*` 若用 Bison
- 中间代码模块：`semantic/`
- 测试用例与输出：`parser/test_syntax_ok.c`、`test_input.c`、运行 `c99_ir.exe` 得到的三地址文本
- 实验报告、PPT
- **AI 辅助**：记录平台、提示词、验证与人工修改（课件硬性要求）

## 5. 一键构建（Windows / MSYS2 MinGW）

在仓库根目录执行 **`build_parse.cmd`**：编译 `lex.yy.o`、`ytab_runtime.o`（**gcc**）、`c99_parse.exe`、`test_lexer.exe`、`c99_ir.exe`、`yacc_subset_scan.exe`，并运行词法/语法/IR 演示。

## 6. 与课件流程图对应关系

- **词法可执行程序**：`test_lexer.exe`（调 `yylex`）
- **语法可执行程序**：`c99_parse.exe`（LALR 总控 + 建 AST）
- **语义引擎（遍历）**：`c99_ir.exe`（三地址骨架）
- **yysemantic.c**：本仓库以 **`semantic/ir_from_ast.cpp`** 承担「第二遍」角色，可与 `reduce_prod` 内嵌语义逐步迁移分工（课件：区分编译期建树与遍历生成 IR）
